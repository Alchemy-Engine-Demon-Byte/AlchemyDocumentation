Resonance

Resonance is the high-performance, thread-agnostic messaging system in Alchemy Engine. It serves as the “nervous system” of the engine, allowing disparate systems (Physics, UI, Gameplay, AI) to communicate without tight coupling.

Resonance is built on top of the Job System and Tag System, making it both highly concurrent and expressively filterable.

Core Concepts

1. Frequency (Channels)

Messages are broadcast on Frequencies, which are represented by Tags (e.g., Game.Event.Explosion). Listeners subscribe to these frequencies using Tag Queries, allowing for powerful hierarchical matching (e.g., listening to Game.Event receives all child events).

2. Payload

Any struct registered with the TypeRegistry can be sent as a payload. Resonance uses a type-erased container (InstancedStruct) to safely copy and pass this data to listeners across threads.

3. Resonate (Broadcast)

“Resonating” is the act of sending a message. This is a non-blocking operation. The message is packaged into a Job and dispatched to the Job System for asynchronous execution.

4. Endpoints

By default, messages are global. Endpoints allow for private, targeted communication. An object (like a UI Window or Entity Component) can create a ResonanceEndpoint and listeners can bind specifically to it, ignoring global traffic.

Usage Guide

Defining a Payload

Any struct can be a payload. It just needs to be reflected.

Listening to a Frequency

To receive messages, you “Listen” using a TagQuery. You also specify where you want the code to run (Main Thread vs Worker Thread) and its priority.

Stopping a Listener

When you no longer need updates, use the handle to stop listening.

Resonating (Broadcasting)

Sending a message is simple and thread-safe.

Advanced: Endpoints (Targeted Messaging)

Endpoints are useful when you want to communicate with a specific instance of a system rather than the whole world. For example, a specific UI Window updating its content.

1. Creating an Endpoint

The owner of the private channel creates an endpoint.

2. Sending to an Endpoint

The sender must have access to the endpoint object.

Debugging

Resonance includes built-in debugging features to help trace message flow.

  • Source Tracking: Every message automatically captures the filename, function, and line number of the sender (Resonate call site).
  • Logging: The system logs registration and dispatch counts to the console.

Architecture & Performance

  • Zero-Copy (Conceptually): The payload is wrapped in a std::shared_ptr when dispatched. If there are 100 listeners, the payload data is NOT copied 100 times; all listeners share read-only access to the same data.
  • Job System Integration: Resonance listeners are executed as individual Jobs. This allows the engine to automatically load-balance message processing across all available CPU cores.
  • Tag Query Optimization: Tag matching is highly optimized using the engine’s token-stream query evaluator.
